[프로젝트 캘린더]
- FullCalendar 라이브러리를 이용해 캘린더 구현 및 프로젝트 전체 기간, 회의일정 이벤트 추가하고,
  이벤트 추가 시 각각 다른 색상으로 지정하여 구분함.
- 프로젝트 전체 기간 클릭 시 프로젝트 Home 화면으로 이동할 수 있도록 기능 구현함.
- 세션으로 로그인한 사용자의 ID 값을 받아 해당 사용자가 참여중인 프로젝트의 정보만 확인할 수 있도록 함.

[회의록]
- 세션으로 로그인한 사용자의 ID 값을 받아 해당 사용자가 참여중인 프로젝트의 회의록만 확인할 수 있도록 함.
- FullCalendar 라이브러리를 이용해 캘린더 구현 및 회의일정 이벤트 추가하였고,
  회의록 등록/미등록 상태에 따라 각각 다른 색상 지정하여 구분함.
- 회의록 등록 상태인 회의 일정 클릭 시 회의록 상세 페이지로 이동하여 회의록 수정 및 삭제 가능.
- 회의록 미등록 상태인 회의 일정 클릭 시 회의록 등록, 회의 일정 수정 및 삭제가 가능한 모달 창 표출시킴.
- 회의록 버튼 클릭 시 캘린더 우측으로 회의록 리스트 표출되도록 ajax 이용해 기능 구현.
- 회의일정 등록 버튼 클릭 시 회의록과 회의 일정을 선택해 등록할 수 있는 모달 창을 표출시켰고,
  회의록으로 등록 시 회의록 등록 상태의 이벤트, 회의 일정으로 등록 시 회의록 미등록 상태의 이벤트 색상으로 입력되도록 함.

[알림]
- 세션으로 로그인한 사용자의 ID 값을 받아 해당 사용자의 권한을 비교해 해당 권한에 맞지 않는 알림칸은 아예 보이지 않도록 처리함.
- WebSocket / Stomp를 이용했으며, 5초 마다 웹소켓 재연결되며 send 처리되어 새로운 알림이 5초마다 실시간 수신될 수 있도록 함.
- 답글 알림 클릭 시 원글과 등록된 답글을 모두 확인할 수 있는 리스트로 이동하여 미확인 상태의 답글 클릭해 확인 시
   alarm_flag가 N으로 변경되면서 알림이 사라질 수 있도록 구현함.
- 댓글 알림 클릭 시 해당 게시글이 팝업으로 표출되면서 해당 게시글에 등록된 댓글의 alarm_flag가 한번에 N으로 모두 변경되어
  알림이 사라질 수 있도록 구현함.
- 로그인한 사용자가 참석자로 지정된 회의 일정 중 접속한 날짜에 등록된 회의 일정이 있다면 알림 표출될 수 있도록 했으며,
   해당 알림 클릭 시 회의록 캘린더 페이지로 이동될 수 있도록 함.
- 팀장 계정으로 접속 시 프로젝트 생성 신청 후 관리자에 의해 승인 처리되면 알림이 발송될 수 있도록 했으며,
  해당 알림 클릭 시 프로젝트 관련 페이지로 이동되고 alarm_flag = N으로 변경될 수 있도록 함.
- 관리자 계정으로 접속 시 미승인 상태인 프로젝트 생성 신청 건수를 알림으로 표출시켰으며,
  해당 알림 클릭 시 프로젝트 생성 승인 처리가 가능한 관리자 페이지로 이동함. (승인 처리 시 prj_approve 값이 변경됨)
